使用Hugo创建自己博客在Github Pages

2019-01-10
hugo

目标

本教程的目标是通过hugo在github上搭建自己的blog

安装hugo

hugo 使用go语言开发的，支持多个平台。最新的release版本可以在Hugo Releases,当然你也可以使用Go编译器工具链源码编译hugo,在http://golang.org/doc/install/source 查看完整的操作系统和编译架构的支持列表。

我是mac OS,使用homebrew安装hugo,只需要运行命令： brew install hugo

生成站点

使用hugo快速生成站点，比如站点目录 ～/Blog

hugo new site ~/Blog

创建文章

创建一篇文章，放在post目录，方便生产聚合页面

hugo new post/markdown.md

安装皮肤

从皮肤列表挑选中意的皮肤，找到皮肤的gitHub地址，然后克隆到themes目录中.

cd themes && git clone https://github.com/xianmin/hugo-theme-jane.git

运行hugo

在你站点的根目录执行 hugo 命令

hugo server --theme jane

在浏览器打开： http://localhost:1313

部署

在github 通过托管个人/组织页面来构建和发布你的github pages站点。我们需要创建两个不同的仓库,一个用于托管hugo的内容，另一个用于托管静态站点(hugo的public文件夹) 。

一步一步来:

  1. 在 GitHub 上创建 <your-project>-hugo 代码仓库（它将用于托管 Hugo 的内容）

  2. 在 GitHub 上创建 <username>.github.io 代码仓库（它将用于托管 public 文件夹：静态的网站）

  3. git clone <<your-project>-hugo-url> && cd <your-project>-hugo

  4. 让你的网站在本地生效（hugo server --watch -t <yourtheme>）

  5. 当你对结果满意时，Ctrl+C (关闭服务)并 rm -rf public（不用着急，使用 hugo -t <yourtheme> 总会重新生成这个文件夹的）

  6. git submodule add git@github.com:<username>/<username>.github.io.git public

  7. 基本上完成：添加 deploy.sh 脚本来帮你（让它可执行：chmod +x deploy.sh）:

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo

# Add changes to git.
git add -A

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master
git subtree push --prefix=public git@github.com:spencerlyon2/hugo_gh_blog.git gh-pages

通过./deploy.sh "Your optional commit message" 把变动推送到 <username>.github.io（小心，你也许想要提交变化到 <you-project>-hugo 仓库）。

就是这样！你的个人页面运行在 http://username.github.io/（在10分钟延迟之后）。
